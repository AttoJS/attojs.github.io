    Service Result Options

# Result

# loading

  • 类型： Ref<boolean>

  • 默认值： false

    Service 请求的执行状态

# data

  • 类型: Ref<R | undefined>

  • 默认值： undefined

    接口返回的数据，如果有传入 formatResultopen in new window，则为格式化后的 data

# error

  • 类型: Ref<Error | undefined>

  • 默认值： undefined

    如果在内部抛出了一个错误，则会被 error 接收并返回

# params

  • 类型: Ref<P[]>

  • 默认值： []

    Service 的请求参数

  • 示例

function getUser(name) {
  return fetch('api/user?' + new URLSearchParams({ name })).then(res => res.json());
}

const { params, run } = useRequest(getUser, {
  defaultParams: ['John'],
});
// 默认请求时，如果存在 defaultParams， 则 params.value 将会等于 defaultParams，否则为空数组

// 当 run 传入参数时，此时的参数将会同步到 params 里面
run('Benny'); // params.value 等于 ['Benny']

# run

  • 类型: (...arg: P[]) => Promise<R>

    手动触发 Service 的请求

    注意

    当开启了 debounceIntervalopen in new window 或者 throttleIntervalopen in new window 模式后 run 的返回结果有可能是 Promise<null>

# cancel

  • 类型: () => void

    • 手动取消本次请求
    • 停止轮询open in new window功能

    注意

    这里说的取消并不是真正的停止请求，只是取消了对 data 的赋值以及 loading 重置为 false 当前的接口请求依旧会继续进行

# refresh

  • 类型: () => Promise<R>

    使用当前的 params 重新调用 run

# mutate

  • 类型: (arg: (oldData: R) => R) => void | (newData: R) => void

    直接修改 data 的结果

  • 参考突变open in new window

# queries

  • 类型: Queries<R, P>

    默认情况下，新请求的数据将会覆盖旧请求的数据。当启用 queryKeyopen in new window 后， queries 能够同时存储多次请求的结果，实现并行的效果

    Queries 的类型如下

    type Queries<R, P> = {
  loading: Ref<boolean>;
  data: Ref<R | undefined>;
  error: Ref<Error | undefined>;
  params: Ref<P[]>;
  run: (...arg: P[]) => Promise<R>;
  cancel: () => void;
  refresh: () => Promise<R>;
  mutate: (arg: (oldData: R) => R) => void | (newData: R) => void;
}
