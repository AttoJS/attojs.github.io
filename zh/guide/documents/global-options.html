# 全局配置

我们通过导出 setGlobalOptions 方法，来给你控制一些配置，避免重复繁琐地填入一些相同的配置

# 基于构建工具

如：vue-cliopen in new window、viteopen in new window

在下面的示例中，当前项目中所有使用到的 VueRequest 都能读取到传入的配置。该配置的作用是把请求的触发方式改为手动触发。更多可配置的全局属性，请参考 globalOptions

// main.ts
import { setGlobalOptions } from 'vue-request';
// ...
setGlobalOptions({
  manual: true,
});
// ...

// App.tsx
import { defineComponent } from 'vue';
import { useRequest } from 'vue-request';
// ...
export default defineComponent({
  name: 'App',
  setup() {
    const { data: user } = useRequest('api/user');
    const { data: job } = useRequest('api/job', { manual: false }); // 覆盖全局配置

    // ...
  },
});

# 使用组件注入配置

除了上述使用 API 的方式去设置全局变量以外，你还可以通过我们导出的 RequestConfig 组件，进行更加精细化的配置。如下所示，可以为不同的组件传入不同的配置。

import { defineComponent } from 'vue';
import { RequestConfig } from 'vue-request';
// ...
export default defineComponent({
  name: 'App',
  setup() {
    return () => (
      <div id="app">
        <RequestConfig config={{ loadingDelay: 300 }}>
          <ComponentA />
        </RequestConfig>
        <RequestConfig config={{ loadingDelay: 500 }}>
          <ComponentB />
        </RequestConfig>
      </div>
    );
  },
});

# 基于 CDN

如果你通过 CDN 来使用 VueRequest 的，你可以通过以下方式来设置全局配置。我们在 VueRequest 的实例上放置了 setGlobalOptions 方法

<!-- ... -->
<script src="https://unpkg.com/vue-request.min.js"></script>
<script>
  VueRequest.setGlobalOptions({
    cacheTime: 12 * 3600,
  });
</script>

提示

配置权重：setGlobalOptions < RequestConfig < 局部 config
