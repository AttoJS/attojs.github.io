# 节流

有关节流的解释，你可以在这里 (opens new window) 找到。

简单来说就是，保证每 X 毫秒去执行一次函数。例如输入事件，throttle 函数会在用户输入的过程中以设定的时间间隔去执行函数。这与我们上一节提到的 debounce 有很大的区别。

注意

VueRequest 的节流是使用 lodash (opens new window) 提供的 throttle (opens new window) 实现的，leading 和 trailing 选项默认为 true，因此，在第一个 throttle 的执行间隔中可能会触发两次请求

如下所示，你只需要提供一个 throttleInterval 来告诉我们节流的执行间隔。这样用户在输入信息时，每隔 300毫秒 我们就会发起一次请求。








 











import { defineComponent } from 'vue';
import { useRequest } from 'vue-request';

export default defineComponent({
  name: 'App',
  setup() {
    const { data, run } = useRequest(name => `api/users?name=${name}`, {
      debounceInterval: 300,
    });
    return () => (
      <div>
        {data.value}
        <br />
        <input type="text" onInput={e => run(e.target.value)} />
      </div>
    );
  },
});
