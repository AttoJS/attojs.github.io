# Refresh On Window Focus

Sometimes, you need to ensure data consistency between multiple browser windows; or when the user's computer is reactivated in the dormant state, the page data needs to be synchronized to the latest state. refreshOnWindowFocus may save you a lot of logic code, as shown below, you only need to enable refreshOnWindowFocus.

Tips

You can try to switch the window or hide the window and then switch back to observe the following example (the default interval is 5 seconds)

import { defineComponent } from 'vue';
import { useRequest } from 'vue-request';
import Mock from 'mockjs';

function testService() {
  return new Promise(resolve => {
    setTimeout(() => {
      resolve(Mock.mock('@name'));
    }, 1000);
  });
}

export default defineComponent({
  name: 'App',
  setup() {
    const { data, loading } = useRequest(testService, {
      refreshOnWindowFocus: true,
    });
    return () => <div>random name: {loading.value ? 'loading...' : data.value}</div>;
  },
});
















 




# RefocusTimespan

If you don’t want users to frequently trigger refreshOnWindowFocus in a short time to cause frequent requests, you can modify [refocusTimespan](../.. /api/index.html#refocustimespan) to extend the trigger interval, as shown below

import { defineComponent } from 'vue';
import { useRequest } from 'vue-request';
import Mock from 'mockjs';

function testService() {
  return new Promise(resolve => {
    setTimeout(() => {
      resolve(Mock.mock('@name'));
    }, 1000);
  });
}

export default defineComponent({
  name: 'App',
  setup() {
    const { data } = useRequest(testService, {
      refreshOnWindowFocus: true,
      refocusTimespan: 2000, // 2000ms
    });
    return () => <div>random name: {loading.value ? 'loading...' : data.value}</div>;
  },
});
