# 什么是 SWR ？

The following passage is taken from vercel/swr (opens new window).

The name “SWR” is derived from stale-while-revalidate, a cache invalidation strategy popularized by HTTP RFC 5861 (opens new window). SWR first returns the data from cache (stale), then sends the fetch request (revalidate), and finally comes with the up-to-date data again.

通过开启 SWR ，vueRequest 首先会在缓存里面获取数据提供给组件，然后再通过接口请求最新的数据。当得到新的数据后，就会把最新的数据重新提供给组件进行渲染。这样对于一些时效性不高的接口，通过使用 SWR 能够使得数据更快地呈现在用户面前，避免了网络条件不佳的用户忍受冗长的 Loading 动画，以获得更好的体验。
